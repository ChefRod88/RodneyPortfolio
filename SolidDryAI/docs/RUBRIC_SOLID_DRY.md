# SOLID + DRY Rubric

## Purpose
This file defines how SolidDry AI evaluates code quality for the SOLID principles and DRY duplication risk.

## Principles Scored
- Single Responsibility Principle (SRP)
- Open/Closed Principle (OCP)
- Liskov Substitution Principle (LSP)
- Interface Segregation Principle (ISP)
- Dependency Inversion Principle (DIP)
- DRY (duplication minimization)

## Scoring Notes
- Each finding must include severity, confidence, and rationale.
- Low-confidence findings must be routed to human review.
- QA adjudication is required for unresolved findings.
