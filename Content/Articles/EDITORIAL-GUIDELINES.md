# Editorial Guidelines

## 1. Audience
The target audience consists of technical founders, engineering managers, and senior developers looking for pragmatic, production-tested solutions in the .NET ecosystem.

## 2. Tone & Voice
- **Professional but direct:** Avoid academic fluff. Get straight to the technical problem and solution.
- **Authoritative:** Speak from experience. Use "I architected" or "We implemented".
- **Business-aligned:** Always tie technical decisions back to business outcomes (e.g., ROI, security compliance, scalability).

## 3. Structure
- **Introduction (The Hook):** Clearly define the problem space.
- **The Architecture/Solution:** High-level overview of the chosen approach.
- **Implementation Details:** Code snippets, configuration, or infrastructure definitions.
- **Trade-offs:** Discuss alternative approaches and why they were rejected.
- **Conclusion:** Summary and call to action.

## 4. Technical Constraints
- All code snippets must compile and be formatted properly.
- Prefer C# 10+ and .NET 9+ syntax.
- Avoid hypothetical examples; use abstracted real-world scenarios.
- All articles must include a `focusKeyword` in the YAML front matter for SEO.

## 5. SEO Requirements
- Titles must be under 60 characters.
- Descriptions must be between 120-155 characters.
- Include the focus keyword in the first paragraph.
- Use logical H2 and H3 headings.
